# XML 注意事项

Hacknet 内使用的 XML 解析器有一定问题，有以下几点需要注意。

## 编码与编码声明

XML 文件首行的 `<?xml version="1.0" encoding="UTF-8"?>` 声明是可选的。根据 Hacknet 源码，只要文件本身采用 UTF-8 编码，省略该行通常不会引发问题。

建议所有内容均使用 UTF-8 编码（不推荐 UTF-8 with BOM 或其他编码），以避免潜在的兼容性问题。

## 标签间隔

Hacknet 的 XML 解析器无法处理紧贴的标签（`<root><foo></foo><bar></bar></root>` 等情况）。

为了防止解析错误，建议尽量保持标签之间至少有一个空格、换行符或缩进符。
