# Hacknet.wiki 贡献指南

感谢你对 Hacknet.wiki 添砖加瓦！

**核心原则：**
- **内容优先**：先写文档，再考虑是否添加新功能。
- **社区驱动**：需要持续优化贡献体验，降低内容贡献的门槛。
- **质量要求**：内容简单清晰准确，最好提供参考来源，使用标准 Markdown 与 VitePress 内置的 [Markdown 扩展](https://vitepress.dev/zh/guide/markdown)。

## 早期阶段须知
目前处于早期开发，**暂时不要**给 Extension Tutorial 贡献新内容。但是我们欢迎对 Extension Tutorial 中已有的内容进行 bug 修复。

## 项目目标

搭建以内容为主，社区驱动的 Hacknet.wiki 文档中心。

## Pull Request 指南

### ✅ 接受的 PR：
- 内容改进（补充、修正、优化）
- 必要的功能更新（需说明理由）
- 小范围修正（拼写、注释等）
- 明确的 Bug 修复

### ❌ 拒绝的 PR：
- 低质或错误内容
- 非必要的功能更新
- 把 Feature 当 Bug “修复”了
- 大规模格式改动

### PR 处理流程：
提交 → 检查 → 修改（如需）→ 测试 → 合并
