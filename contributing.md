# Hacknet.wiki 贡献指南

感谢你为 Hacknet.wiki 添砖加瓦！

Hacknet.wiki 以 **内容** 为主，以 **由社区驱动** 为目标，
因此项目结构会尽可能让 **内容贡献** 变得更简单。

在提交你的贡献前，请确保你已经阅读了以下内容：
- [Hacknet.wiki 项目目标](#hacknetwiki-项目目标)
- [行为准则](./code_of_conduct.md)
- [Pull Request 指南](#pull-request-指南)

## Hacknet.wiki 项目目标

>TL;DR:
Hacknet.wiki 是一个内容优先、社区驱动的文档站。\
\
内容为主：优先更新和改进文档内容，功能更新仅在必要时进行（需显著提升效率且无重大缺点）。\
社区驱动：优化贡献体验，降低门槛，同时保持项目结构稳定。\
文档站目标：提供清晰、准确的参考信息，减少错误。

Hacknet.wiki 以 **内容** 为主，以 **由社区驱动** 为目标的文档站。

### 以 内容 为主

Hacknet.wiki 是一个文档站，那么就会有两种更新：
- 内容更新：改变文档内容
- 功能更新：改变文档站的网站功能。

以 内容 为主意味着添加或修改内容是最重要的，也是首要目标，其余次之。

Hacknet.wiki 基于 vitepress，具备很多开箱即用功能，被认为是已经满足大部分需求。在这个情况下，应尽可能减少功能更新。

除非有正当理由认为这个更新是必要的——能够提高使用效率，且没有缺点或缺点极小的更新。

### 以 由社区驱动 为目标

意味着 Hacknet.wiki 需要优化贡献体验，在尽可能不破坏项目结构的情况下降低贡献门槛。

### 文档站

文档站目的在于指导他人、供人参考，那么就需要做到：
- 尽可能言简易懂、解释清楚
- 尽可能减少错误
- 最好提供对应内容的参考来源（本站链接、OpenHacknet对应源码链接、官方教程链接、hnxml链接等）
- 使用合法的 Markdown 语法与 vitepress 内置的 [Markdown 扩展](https://vitepress.dev/zh/guide/markdown)。格式不作要求，只要不要太离谱就行。
- 除了自动生成的文件，保持其他文件末尾空一行。

## Pull Request 指南

### 哪些 Pull Request 会被通过？

**TL;DR: 保持内容优质，避免无关变更。**

**✅ 会被通过的 PR：**  
- **内容改进**：添加遗漏内容、修正错误、优化解释（需正确且有用，符合 [文档站](#文档站) 对内容的要求）。
- **必要功能更新**：需明确说明为何项目需要（符合 [以内容为主](#以-内容-为主) 原则）。
- **小修小补**：拼写/注释修正等（尽量合并提交，避免无关格式改动）。
- **明确的 Bug 修复**：确认为 bug 的问题修复。

**❌ 不会被通过的 PR：**  
- 低质内容（错误、废话等）。
- 非必要功能更新（违背项目目标）。
- 误将功能当 bug 修复。
- 大规模或无关紧要的格式修改。
